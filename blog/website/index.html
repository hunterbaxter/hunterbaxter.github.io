    • Constraints for framework choice

    • must support markdown for writing content
    • enough good looking themes to give me guidance on how to build my own
    • written in a language I know well in case I theoretically needed to fix something into source code (python and/or Rust)
    • single binary, and fast live updates when modifying site locally to speed up development
    • templating language like Jinja2 (ansible's)

    The reason I came up with these constraints is through using Jekyll and Hugo for previous website iterations, and being unhappy with them. The issues I found were:

    • Jekll had an unpleasant file structure, and the theme I was using was frankly too complicated for my needs
    • Hugo has great themes, but I found customizing them difficult due to confusing/complicated templating language, and the theme writers being much more talented front-end developers, so I didn't understand the work.
    • I found customizing image display in a responsive manner difficult with Hugo

    So, after some research, I landed on Zola, which satisfies all of my constraints. The downside of Zola is that its documentation is much more sparse, and there are not nearly as many themes. However, after evaluating the themes out there, I realized I wanted something custom anyways (and I wanted to learn basic HTML and CSS).

    Design

    For my design, I wanted to have four main sections of the website:

    1. A more artsy homepage that has a gallyery of pictures (not implemented)
    2. An about page what consists of a brief profile about who I am (professionally and briefly personally)
    3. A contact page which consists of a way to contact me and view my public accounts (email, pgp, linkedin)
    4. When I have more publications, a page where I can list them and provide pdf to them
    5. A page where I can have posts which contain content I don't mind sharing to the public (if I spend time doing / creating something, I want others to be able to replicate it without much effort).
