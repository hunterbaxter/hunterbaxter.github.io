    • Constraints for framework choice

    • must support markdown for content (code snippets, math)
    • enough themes where I can get example html/css from
    • written in a language I know well in case I theoretically needed to fix something (python and/or Rust)
    • single binary, and fast live updates when changing to speed up development
    • simple file structure
    • templating language like Jinja2

    features of site (framework independent)

    • mobile friendly
    • colorscheme switcher (pretty much everything can do this with simple css)
    • rich markdown (code snippets, latex, references, table of contents)

    The reason I came up with these constraints is through using Jekyll and Hugo for previous website iterations. I originally used a premade theme with barely any customization with Jekyll, but I found that I had too many files, and it was just unpleasant to use (opinion, ones experience may vary - I am inexperienced with front end). Hugo was almost exactly what I wanted, but I ran into two main issues: (1) Customization is difficult due to confusing/complicated templating language (I am only familiar with Jinja2), which made updating a theme that was pretty good very confusing. (2) I found customizing image display in a responsive manner difficult (again, ones experience may vary, I am inexperienced with front-end)

    So, after some research, I landed on Zola, which satisfies all of my constraints. The downside of Zola is that its documentation is much more sparse, and there are not nearly as many themes. However, after evaluating the themes out there, I realized I wanted something custom anyways (and I wanted to learn basic HTML and CSS).

    Design

    For my design, I wanted to have four main sections of the website: (1) A more artsy homepage that has my face, as well as buttons to access other parts of the site (2) An about page what consists of a brief profile about who I am (professionally and briefly personally) (3) A contact page which consists of a way to contact me and view my public accounts (email, pgp, linkedin) (4) When I have more publications, a page where I can list them and provide pdf to them (5) A page where I can have posts which contain content I don't mind sharing to the public (if I spend time doing / creating something, I want others to be able to replicate it without much effort).
